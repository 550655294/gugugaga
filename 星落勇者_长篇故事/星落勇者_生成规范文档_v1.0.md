## 一、世界观

### 🌍 艾斯特拉大陆（Astra Continent）

千年一遇的**星落之夜**，星辰碎片自天穹坠落，选中凡世生灵成为「星选勇者」。
这一次，星辰坠入极北的人类边陲村落，选中了村中唯一的企鹅少女。

## 五、对话系统（咕嘎 + 叮咚翻译 + 史诗旁白）

### 🔑 核心规则

> 妹妹**只说"咕嘎"变体**，叮咚**翻译成人话**。两者的台词在提示词中成对出现。
> 每集穿插**史诗旁白**（日文或英文），增强世界观的厚重感与仪式感。

### 🎙️ 旁白（Narrator）— 星辰之声

| 属性 | 设定 |
|------|------|
| 身份 | 无名叙述者，如同星空中回荡的古老声音，讲述艾斯特拉大陆的命运 |
| 声线 | 深沉浑厚的男声（后期配音用），如纪录片旁白或动画预告解说 |
| 语言 | **日文**（主，动漫史诗感）或 **英文**（辅，国际化），每集选用一种 |
| 出现时机 | ① 开场白（每集必有）② 关键转折（1-2次/集）③ 片尾结语 |
| 风格 | 短促有力、充满仪式感——像《进击的巨人》旁白或《最终幻想》开场白 |
| 字数 | 每段旁白 ≤ 30 字（中文），≤ 20 词（日/英） |

### 旁白出现时机

| 时机 | 位置 | 示例（日文） | 示例（中文意译） |
|------|------|------|------|
| 📖 开场白 | 场景一开头 | `星は、落ちた。` | 「星辰，坠落了。」 |
| ⚡ 关键转折 | 敌人登场/力量觉醒/绝境反转 | `その小さき翼が、運命を変える。` | 「那双小小的翅膀，将改变命运。」 |
| 🔚 片尾结语 | 收尾段末 | `かくして、勇者たちの旅は始まった。` | 「于是，勇者们的旅途开始了。」 |

### 旁白语言选择规则

> 每集**固定选用一种**，不在同一集中混用日文和英文旁白。

| 集数 | 推荐语言 | 理由 |
|:--:|:--:|------|
| 1 | 日文 | 开篇用日文奠定史诗基调 |
| 2-5 | 英文/日文交替 | 冒险展开，英/日轮换保持新鲜感 |
| 6 | 日文 | 古龙遗迹·传承回，日文仪式感最强 |
| 7-9 | 日文 | 高潮段落，日文张力最足 |
| 10 | 英文 | 终章用英文收束，国际化的告别 |

### 对话格式模板（含旁白）

```
【旁白】日: 『星は、落ちた。』  / 英: "The star has fallen."

妹妹[动作+表情]：「咕嘎！」
叮咚眨动金眼：「[翻译内容]」
```

### 翻译风格

| 妹妹的咕嘎变体 | 情绪 | 叮咚翻译示例 |
|------|------|------|
| 咕嘎！ | 坚定/宣告 | 「放马过来。」「我们上。」 |
| 咕嘎咕嘎！ | 兴奋/赞同 | 「太棒了！」「就是这个！」 |
| 咕……嘎…… | 委屈/失落 | 「好累……」「为什么……」 |
| 咕嘎？ | 疑问 | 「什么意思？」「你是谁？」 |
| 咕嘎~~~ | 满足/幸福 | 「好舒服~」「赢了~」 |
| 咕嘎！咕嘎！ | 着急/慌乱 | 「快跑！」「来不及了！」 |

### 叮咚的独立台词

叮咚不仅翻译，偶尔有自己的吐槽/解说（每集≤3次独立发言，其余是翻译）：

| 场景 | 叮咚独立发言示例 |
|------|------|
| 妹妹莽撞冲进敌群 | 「唉，她又不等我说完冲了。」
| Doro面无表情一击必杀 | 「……有时候真的不知道她在想什么。」
| 菲比施法翻车炸了自己 | 「菲比小姐的魔法事故，本集第三次了。」
| 战斗胜利 | 「冰、炎、光——三人齐了，还有谁不服？」

### 场景时长弹性规则

> ⚠️ 每个场景不强制 12s。根据节奏选择 6s/9s/12s，由内容决定时长，不是由时长决定内容。

| 场景类型 | 段数 × 3s | 总时长 | 适合场景 |

| 原则 | 说明 |

## 十、即梦生成参数（奇幻版）

```
模型：Seedance 2.0 Fast VIP | 参考模式：全能参考
比例：9:16（竖屏） | 时长：6/9/12s × 5-7段 = 60-85s/集 | 输入：图生视频(角色参考图+英文提示词)

积分预估：5-7段 × 200 ≈ 1000-1400积分
提示词长度：35-80 英文单词（英文段）
迭代策略：一次只改镜头/动作/光线其中一个变量
```

### 🎬 @图N引用语法（即梦全能参考模式）

> ⚠️ 即梦 Seedance 2.0「全能参考」模式下，提示词中必须使用 `@图N 用途` 格式告知 AI 每张上传图片的用途。上传顺序即 @图N 编号——第一张上传的是 @图1，第二张是 @图2，以此类推。

#### 语法格式

```
@图N 用途描述
```

| 用途类型 | 格式 | 含义 | 何时使用 |
|----------|------|------|----------|
| 角色参考 | `@图N参考[角色名]外观` | 锚定角色外观（面部、体型、服装） | 该角色在本场景出镜时必传 |
| 首帧 | `@图N作为首帧` | 以此为视频起始画面 | 每个场景必传首帧图 |
| 道具参考 | `@图N参考[道具名]外观` | 锚定道具形状材质 | 关键道具（剑/法杖等）首次或持续出现时 |
| 场景参考 | `@图N参考场景环境` | 锚定背景风格光照 | 需固定环境风格时 |
| NPC参考 | `@图N参考[类型]外观` | 锚定群像/次要角色 | 村民/士兵等非主线角色 |

#### 编号对齐铁律

```
⚠️ 即梦操作指南中「@图」列表的从上到下顺序 = 即梦上传顺序 = 提示词中 @图N 的编号。
标题改为「@图 | 来源 | 用途」三列，第一列标注 @图N，第二列标注素材编号(①②③...)。
```

#### 示例：场景3（握剑·叮咚觉醒）

| @图 | 来源 | 用途 |
|:--:|:--:|------|
| @图1 | ① | 角色参考：妹妹·普通外观 |
| @图2 | 📸 | 首帧：场景2尾帧截图 |
| @图3 | ③ | 角色参考：叮咚 |
| @图4 | ④ | 道具参考：冰晶之剑 |

对应提示词：
```
📎 @图1参考妹妹外观 @图2作为首帧 @图3参考叮咚外观 @图4参考冰剑外观

（0-3s）：@图1参考妹妹外观 @图2作为首帧 @图3参考叮咚外观 @图4参考冰剑外观 奇幻萌系Q版...
```

#### 双外观切换时的 @图N 标注

> ⚠️ 当角色有多个外观变体（如妹妹·普通外观① vs 妹妹·勇者外观②），切换场景时：
> - @图1 标注改为 `@图1参考妹妹勇者外观`（加"勇者"二字区分）
> - 即梦操作指南表格中 @图1 的来源列标注为 ②

| 场景 | @图1 写法 | 原因 |
|------|-----------|------|
| 场景1-4 | `@图1参考妹妹外观` | 普通外观（①），无披风无剑 |
| 场景5+ | `@图1参考妹妹勇者外观` | 勇者外观（②），披风+持剑 |
