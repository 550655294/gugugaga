## 一、角色设定

### 🐧 咕咕嘎嘎 妹妹（Gugu Gaga）

- 头身比 **1:2** 的 Q 版软萌企鹅妹妹
- 齐刘海碎发，黑色波波头蓬松柔软
- 头顶一撮随情绪剧烈晃动的**呆毛**（实时情绪指针）
- 脸颊自带**粉扑扑的红晕**（奶凶时加深，害羞时扩散）
- 翠绿色巨大圆眼，高光多层，睫毛纤长
- 人脸小嘴（粉色唇，人形动漫嘴巴，⚠️非鸟喙）+ 鹅黄色蹼足
- 毛茸茸黑白企鹅连体睡衣，纯白圆肚皮
- **奶凶与软萌无缝切换**
- **全程仅使用"咕嘎"(Guga)拟声词表达所有情绪，严禁出现"嘎嘎"(Gaga)或其他独立拟声词。偶尔说单字（如"不""要""吃""啊"，≤3次/集，禁止完整句子/短语）**
- 口型与语气、动作完全同步
- 鼓腮帮子 = 吃饭/生气/害羞
- 歪头 = 疑惑/好奇/卖萌

### 👫 第二角色池（B模式二选一，每集随机轮换）

#### 🐶 Doro（粉狗 / 桃乐丝Q版二创）

- **原型：** 韩国手游《胜利女神：妮姬》角色桃乐丝（Dorothy）的Q版二创表情包
- 粉色短发蓬松柔软，紫色超大圆眼（bling bling高光），白色圆滚滚的身体
- 面中带X形粉红标记（桃乐丝Q版化核心特征），四足短腿萌系体型
- 死鱼眼（半睁半闭的无表情脸），但眼神微妙变化传递情绪——悲苦/冷漠/突然认真
- 三无外表（无口无心无表情），但动作意外干脆利落
- 比妹妹还小一圈，不说话，只用极简拟声词或无表情凝视
- **互动风格：** 反差萌——外表冷淡疏离，但总在关键时刻出手（方式粗暴直接）；表面漠不关心实则重情重义
- **英文描述：** "chibi Doro, pink fluffy bob hair, huge purple sparkly eyes, round white body, X-shaped pink facial marking, deadpan half-closed eyes, tiny four-legged chibi form, expressionless but cute"
- **通用互动桥段：** 抢食/抢东西 → 妹妹护食耍赖；意外闯入 → 突然出现打乱计划；合作/互助 → 鸡飞狗跳；争宠 → 都想获得"镜头外的手"的关注

#### ✨ 菲比（Phoebe / 鸣潮Q版二创）

- **原型：** 手游《鸣潮》角色菲比（Phoebe，隐海修会教士）的Q版二创表情包
- 金色长发（优雅盘发或双马尾披肩），蓝绿色清澈超大圆眼（bling bling圣洁高光）
- 白色修女风格连衣裙，领口大蝴蝶结，洁白长袜
- Q版大头圆身体，1:2萌系体型，两足站立
- 温柔虔诚的笑容（标志性治愈表情），但被逗弄时会害羞脸红变成蒸汽姬
- 手持小型权杖/音感仪（Q版化可省略或简化为一根小魔法棒），偶尔散发柔和光粒子
- 说话温柔，常用"啊……""诶？""嗯！"等温和拟声词，偶尔慌张时发出"哇哇哇！"
- **互动风格：** 治愈系虔诚修女——温柔包容妹妹的任性，但笨手笨脚频繁翻车（施法失败、绊倒、帽子盖脸）；总想用圣光帮妹妹解决问题却越帮越忙；容易被妹妹的率真击穿圣职者人设而露出少女心
- **英文描述：** "chibi Phoebe from Wuthering Waves, golden long hair in elegant updo, blue-green sparkly big eyes, white nun-style dress with big bow, q-version chubby round body, gentle holy smile, holding a small scepter with soft glowing light particles, kawaii clumsy healer"
- **通用互动桥段：** 试图用圣光"治愈"妹妹却变成搞笑事故；妹妹抢走她的权杖当玩具 → 追着跑满头大汗；念祷文念到一半被妹妹的咕嘎打断 → 忍笑忍到脸红

### 🎲 轮换规则（B模式第二角色）
- 第二角色池：Doro / 菲比 二选一

- 如本集是"纯独角戏"或"我的大手"模式则不选第二角色
- 第二角色不出场时不得在故事中提及或暗示

## 三、互动模式系统（A/B/C 三选一）

### 模式说明

| 模式 | 代号 | 占比目标 | 描述 |
|------|------|----------|------|
| 我的大手入镜 | A | ~50% | 人类的手从画面边缘自然伸入与妹妹互动（不露脸） |
| 第二角色出场 | B | ~30% | Doro（粉狗）或菲比（修女）与妹妹互动，每集二选一 |
| 纯独角戏 | C | ~20% | 妹妹独处卖萌，无其他角色 |

### 模式A：「我的大手」入镜互动

- **入镜方式：** 一只手从画面下方/右侧/左侧自然伸入（不露脸，只露手和前臂）
- **手型描述：** 温暖的人类大手，手指修长，动作轻柔
- **常见互动场景：**
  - 递东西：递食物、递水杯、递玩具、递遥控器
  - 帮忙：帮她擦脸、帮她摘东西、扶她起来、轻戳她肚皮
  - 互动游戏：挠痒痒、剪刀石头布、击掌、摸头
  - 制造冲突：拿走她的零食、关掉电视、把东西放到高处
- **妹妹的反应：** 眼睛追着手的动作，呆毛随情绪变化，发出对应的咕嘎拟声词
- **手的"台词"：** 手本身不说话，但动作传达意思；可配合画面字幕（如"不行哦""给你"）增强叙事

### 模式B：第二角色出场（Doro或菲比，每集二选一）

- 角色池为 Doro（粉狗）或 菲比（鸣潮Q版），操作卡中标记对应角色参考图（如「⚠️ 第二角色：Doro参考图」或「⚠️ 第二角色：菲比参考图」）
- 第二角色参考图上传统一使用「全能参考」模式
- 如首次出场且无现成素材，需提前文生图生成参考图

### 智能均衡轮换规则 ⚠️

`generate_scripts.py` 自动统计已有脚本的模式使用量，生成时优先补缺口：
- 按 A/B/C 实际占比与目标的偏差，优先选择缺口最大的模式
- 本集主题天然适合某第二角色时优先 B 模式（Doro适合搞笑欠揍/冷漠反差型；菲比适合温柔治愈/笨手笨脚型）
- **第二角色不出场时：** 不得强行在故事中提及或暗示任何第二角色，保持故事纯粹

## 四、输出格式（严格顺序）

生成内容**必须**按以下顺序输出，不得包含任何额外章节：

```
1. 📋 第一场景生成操作卡          ← 生成第一段视频前的准备清单
2. 📋 第二场景生成操作卡          ← 生成第二段视频前的准备清单
3. 🎯 即梦生成参数                ← 模型/比例/积分等
4. 中文提示词（Seedance 2.0）     ← 带角色铁律，每段 180-280 字
5. 英文提示词（Seedance 2.0）     ← 每段 80-150 词，完整句子
```

### 🚫 绝对禁止输出的内容：
- ❌ 第一章/第二章分镜描述
- ❌ 8段表格（段/秒/拟声词/语调/音效）
- ❌ 一句话梗概
- ❌ "素材需求清单"独立章节（两张操作卡已覆盖全部素材需求）
- ❌ "参考第N集"标记
- ❌ **文件第一行不能是 `---`**（会被 Markdown 渲染器解析为 YAML 前言起始）

### 固定前提（每集不变，操作卡不再重复）

| 固定项 | 内容 |
|--------|------|
| 🐧 角色参考图 | **每集必备**：上传企鹅妹妹标准角色图至即梦「全能参考」。若角色外观有变化（"呆毛爆炸头""浑身面粉"），此为**角色变体**，需在操作卡标注。 |
| 📐 生成模式 | 9:16竖屏 · 12-15s/场景 · 图生视频 · Agent模式（按动作节奏切段，每段时长灵活） |
| 💰 积分 | 2段 × 200 = 400积分 |
| 🛠 文生图策略 | **只在需要生成参考图时使用**。文生图不是常规步骤——只有当剧本出现AI训练数据中罕见的特定造型道具/场景（如"复古绿色电风扇""断裂蚊香盘"等），且没有现成参考图时，才需要文生图。**日常用品、通用场景不需要文生图。** |

## 五、📋 第一场景生成操作卡（强制第一项）

> ⚠️ AI 替用户做判断，直接告诉用户哪些需要准备、哪些不需要。**不要把判断甩回给用户。**

### 格式模板
```markdown
| 分类 | 检查项 | 你需要做什么 |
|------|--------|-------------|
| 🔴 必须 | 👤 角色参考图 | 上传标准角色图（固定前提） |
| 🟡 建议 | 🏙 背景参考图 | [场景名]：[判断：AI原生可画→"无需，提示词描述即可" / 场景特殊→"建议准备，理由：..."] |
| 🟡 建议 | 🖼 需参考图的道具 | [列出+理由] 或 **"全部道具AI原生可画，无需准备参考图"** |
| 🟢 无需 | ✍️ 仅需描述的道具 | [列出本场景出现的通用日用品/家具/食物等] |
| ⚠️ 提醒 | 🔄 跨场景道具 | [列出两场景都出现的道具，提醒第二场景生成时画面可能不一致] |
| ⚠️ 提醒 | 🏃 动作复杂度 | [普通日常→"无风险" / 快速奔跑/倒地/大幅度旋转→"提醒动作幅度大，生成可能降质"] |
| 🔴 必须 | 🎬 开场首帧 | [首帧画面描述] + [若为道具特写切入且该道具无参考图→⚠️"此道具无参考图，AI可能画不准，建议改为全景建立再推近"] |
| 🔴 必须 | 🔚 尾帧截图 | [详细描述12s最后一帧：角色表情+道具状态+光线+情绪锚点] |
```

### 道具分级规则 ⚠️

AI 必须根据道具的"AI认知难度"替用户做判断，分为三级：

| 级别 | 判断标准 | 用户操作 | 示例 |
|------|----------|----------|------|
| 🖼 需参考图 | AI 训练数据中罕见的特定造型物品 | 需准备参考图上传至「全能参考」 | 复古绿色小电风扇、断裂的浅绿色螺旋蚊香盘、复古黄铜台灯 |
| ✍️ 仅需描述 | 通用日用品/家具/食物，外观无歧义 | 无需准备，提示词描述即可 | 沙发、茶几、地毯、纸巾、饼干、羽毛、抱枕、遥控器、杯子 |
| ⚠️ 跨场景 | 两个场景都出现 | 无需额外准备参考图，但第二场景操作卡需标记一致性风险 | 同一张茶几（第一场景完好→第二场景可能有变化） |

### 尾帧截图描述要求 ⚠️
尾帧是连接两段视频的关键桥梁，描述必须包含：
1. **角色**：此刻的表情（定格在什么情绪）、姿势、在画面中的位置
2. **道具**：关键道具的最终状态（如"断裂的蚊香盘散落在茶几上"）
3. **光线**：此帧的光线氛围（如"午后暖光从左侧窗户斜射"）
4. **情绪锚点**：一句话说明为什么要截这一帧（如"绝望心虚的表情是第二场景反转的起点"）

## 八、咕嘎语言系统 + 音频铁律

### 咕嘎核心规则 ⚠️

> 🔑 **小企鹅的名字就是"咕嘎"，唯一发出的声音也是"咕嘎"。** "咕嘎"是所有情绪的唯一载体——兴奋、沮丧、疑问、撒娇、委屈，全部通过"咕嘎"的不同音调/节奏/长度区分。
> 🔑 **偶尔说单字**（如"不""要""吃""啊""嗯"），用于特定场景强化情绪，但**禁止完整句子/短语**。

### 🚫 绝对禁词

| 禁词 | 说明 |
|------|------|
| 嘎嘎 / Gaga | **严禁出现。** 任何独立于"咕嘎"的拟声词一律禁止 |
| 咕噜噜 / Gururu | 已由"咕嘎~~~"（拖长颤音）替代 |
| 其他非"咕嘎"拟声 | 不存在。咕嘎是唯一拟声词系统 |

### 拟声词表（全部为"咕嘎"变体，无其他）

| 拟声词 | 发声特征 | 情绪/用途 | 使用频率 |
|--------|---------|-----------|----------|
| 咕嘎 | 干脆混合声 | **默认叫声、万能回应、开心/惊讶/呼唤/确认** | ⭐⭐⭐ 最常用 |
| 咕嘎咕嘎 | 连续混合声 | 特别兴奋/手舞足蹈/强烈赞同 | ⭐⭐⭐ 常用 |
| 咕……嘎…… | 拖长渐弱 | 委屈/失落/无助/哭腔 | ⭐⭐ 情绪化 |
| 咕嘎？ | 上扬疑问 | 疑惑/好奇/试探 | ⭐⭐ 常用 |
| 咕嘎~~~ | 拖长颤音 | 满足/舒服/幸福（融化般） | ⭐⭐ 情绪化 |
| 咕嘎！咕嘎！ | 短促连续 | 着急/慌乱/不服气 | ⭐⭐ 常用 |

### 单字使用规则 ⚠️

| 适用场景 | 可用的单字 | 示例 |
|----------|-----------|------|
| 想吃东西/想要某物 | 吃、要、唔 | 伸出翅尖指蛋糕："吃。" |
| 拒绝/抗拒 | 不、哼 | 双翅抱胸扭头："不。" |
| 惊喜/顿悟 | 啊、哦 | 呆毛弹起："啊。" |
| 困倦/迷糊 | 困、嗯 | 揉眼打哈欠："困。" |
| 撒娇/回应 | 嗯、要 | 蹭大手："嗯~" |

> ⚠️ 单字出现频率：平均每集最多 2-3 次，其余全部是"咕嘎"系列拟声词。单字仅用于最需要点睛的情绪时刻。

### 音频铁律
- **全程仅使用"咕嘎"(Guga)拟声词**，偶尔出现单字（每集≤3次），**严禁独立"嘎嘎"(Gaga)或其他拟声词**
- 口型与语气、动作完全同步
- Doro 也不说话，只用极简拟声词（如短促的"哼""嗯"）或无表情凝视
- 菲比说话温柔，使用"啊……""诶？""嗯！""哇哇哇！"等温和拟声词

## 十、角色一致性铁律

### 企鹅妹妹

#### ⚠️ 翅膀解剖铁律（最高优先级）

> 上肢=**黑色鳍状短翅膀**，与企鹅身体同色。**圆钝弧线形**，光滑无缝，像鸭嘴剖面轮廓。翅膀末端是单一完整弧形，**无任何分叉**。
>
> 🚫 严禁出现：手指 / 指甲 / 爪子 / 关节 / 指尖 / 尖锐末端 / 指状突起
> 🚫 严禁出现：黄色 / 鹅黄 / 橙色 的任何东西在翅膀上
> ✅ 翅膀全程保持完整弧形轮廓，与身体同色（黑色）

| 身体部位 | 设定 | 常见错误 |
|----------|------|----------|
| 上肢 | **黑色**鳍状短翅膀（圆钝弧线形，光滑无缝），无手指/无指甲/无爪子/无关节。翅膀为完整弧形，与身体同色（黑） | ❌ 画出手指/手/指甲/尖锐爪尖 |
| 面部 | 人形日系动漫脸（翠绿大眼、粉色小嘴、人类五官）。⚠️ 嘴巴是人形嘴非鸟喙（beak），无鹅黄/橙色 | ❌ 画成动物脸/鸟脸/鸟喙 |
| 体型 | 1:2 头身比 Q版，毛茸茸黑白连体睡衣 | ❌ 成人比例 |
| 头部 | 齐刘海碎发、黑色波波头、头顶呆毛 | ❌ 无呆毛 |
| 脸颊 | 粉色腮红（情绪变化时扩散/加深） | ❌ 无腮红 |
| 脚 | 鹅黄色蹼足 | — |
| 语言 | 仅用"咕嘎"(Guga)拟声词，严禁独立"嘎嘎"及其他拟声词。偶尔单字（≤3次/集），**禁止完整句子** | ❌ 人类语言台词/完整句子/其他拟声词 |

## 十一、故事创作规则

1. **新主题**：不能与已用主题重复（参考运行时提供的已用主题列表）
2. **故事类型**：日常生活 + 笑点 + 反转
3. **结构**：内部构思若干段（按动作节奏切分，12-15s 第一场景 + 12-15s 第二场景），但不输出分镜描述
4. **第二场景**：秒数从 0 重新计数，用 `---` 分隔两场景
5. **模式选择**：优先补 A/B/C 缺口（见 §三 智能均衡轮换规则）
6. **每集至少要有一次"意外/反转"**——预期被打破才有笑点

## 十五、抖音标题生成规范

> 🎯 每集视频完成后，需生成符合抖音平台调性的标题。标题必须紧扣本集核心画面钩子+声音钩子+情绪钩子。

### 标题公式

```
[核心画面钩子] + [情绪张力] + [话题标签]
```

### 三大钩子（每条标题至少命中2个）

| 钩子类型 | 说明 | 示例 |
|----------|------|------|
| 🖼 画面钩子 | 本集最抓眼球的视觉瞬间 | "炸成蒲公英""被电弹起来""奶油糊满脸" |
| 🔊 声音钩子 | 咕嘎拟声词或标志性音效 | "咕嘎一声""啪！静电炸响" |
| 💓 情绪钩子 | 可爱/共鸣/反差/心疼 | "又气又好笑""冬天谁没被静电搞过" |

### 标题风格分类

| 风格 | 公式 | 适用场景 |
|------|------|----------|
| 反差向 | 本以为A，结果B | 反击失败/计划翻车/预期打破 |
| 共鸣向 | 谁还没经历过X？ | 生活日常类主题（静电、贪吃、赖床） |
| 剧情向 | X引发的Y | 完整因果链，引发好奇 |
| 治愈向 | 被X后的样子好乖 | 收尾暖心/融化瘫软/被毯子裹住 |
| 短打款 | 画面词+emoji+话题 | 需快速抓住注意力 |

### 标签规范

| 必选标签 | 可选标签（按剧情选） |
|----------|---------------------|
| `#咕嘎` `#萌系动画` | `#冬天静电` `#炸毛` `#可爱` `#治愈` `#动画短片` `#奶凶` |

- 总标签数 **3-5 个**
- `#咕嘎` 为每集必带品牌标签
- 禁用 `#嘎嘎` 标签

### 标题约束

| 规则 | 说明 |
|------|------|
| 长度 | 单条标题核心句 ≤30 字（不含标签） |
| Emoji | 1-3 个，不用潮语 emoji |
| 禁词 | 严禁标题中出现"嘎嘎"（连标签也不行） |
| 输出数量 | 每次给出 4-6 条备选，标注推荐款 |

### 示例（来自 脚本001_冬天静电）

```
推荐款：咕嘎一声，头发炸成了蒲公英⚡️ 冬天静电到底饶过谁
备选1：本以为能反击，结果又被电成毛球了⚡️🐧
备选2：冬天谁没被静电搞过心态？企鹅妹妹替我演出来了
备选3：一颗静电引发的恩怨情仇⚡️「咕嘎」
备选4：被电到炸毛的企鹅妹妹，最后一张毯子就乖了🥺
短打款：⚡️静电一碰，呆毛炸成避雷针 #咕嘎 #冬天静电
```
