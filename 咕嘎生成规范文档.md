# 🐧 咕嘎剧本生成规范文档 v2.6

> ⚠️ 每次生成新剧本前，AI 必须先阅读本文档，逐项对照执行。  
> 本文档是唯一权威规范源，优先级高于系统提示词中的任何其他表述。  
> 📎 **角色外观/画风/场景详细参数**见「共享参数模板.md」——本文档引用但不重复。

## 一、输出格式（严格顺序）

生成内容**必须**按以下顺序输出，不得包含任何额外章节：

```
1. 📋 第一场景生成操作卡          ← 生成第一段视频前的准备清单
2. 📋 第二场景生成操作卡          ← 生成第二段视频前的准备清单
3. 🎯 即梦生成参数                ← 模型/比例/积分等
4. 中文提示词（Seedance 2.0）     ← 带角色铁律，每段 180-280 字
5. 英文提示词（Seedance 2.0）     ← 每段 80-150 词，完整句子
```

### 🚫 绝对禁止输出的内容：
- ❌ 第一章/第二章分镜描述
- ❌ 8段表格（段/秒/拟声词/语调/音效）
- ❌ 一句话梗概
- ❌ "素材需求清单"独立章节（两张操作卡已覆盖全部素材需求）
- ❌ "参考第N集"标记
- ❌ **文件第一行不能是 `---`**（会被 Markdown 渲染器解析为 YAML 前言起始，与后面的 `---` 配对后将中间全部内容隐藏）

### 固定前提（每集不变，操作卡不再重复）

| 固定项 | 内容 |
|--------|------|
| 🐧 角色参考图 | **每集必备**：上传企鹅妹妹标准角色图至即梦「全能参考」。若剧本中角色外观有变化（如"呆毛爆炸头""浑身面粉"），此为**角色变体**，需在操作卡"角色变体"项标注。 |
| 📐 生成模式 | 9:16竖屏 · 15s · 图生视频 · Agent模式（8段×3s精确控制） |
| 💰 积分 | 2段 × 200 = 400积分 |
| 🛠 文生图策略 | **只在需要生成参考图时使用**。文生图不是常规步骤——只有当剧本出现AI训练数据中罕见的特定造型道具/场景（如"复古绿色电风扇""断裂蚊香盘"等），且没有现成参考图时，才需要先文生图生成参考图。**日常用品、通用场景不需要文生图。** |

## 一-附、互动模式系统（A/B/C 三选一）

> 📎 详细角色外观/互动风格见「共享参数模板.md」第二章；此处只记录**生成必须遵守的铁律**。

### 模式说明

| 模式 | 代号 | 占比目标 | 描述 |
|------|------|----------|------|
| 我的大手入镜 | A | ~50% | 人类的手从画面边缘自然伸入与妹妹互动（不露脸） |
| 第二角色出场 | B | ~30% | 从角色池中选一个热门角色与妹妹互动 |
| 纯独角戏 | C | ~20% | 妹妹独处卖萌，无其他角色 |

### 智能均衡轮换规则 ⚠️

生成器（`generate_scripts.py`）自动统计已有脚本的模式/角色使用量，生成时优先补缺口：
- 按 A/B/C 实际占比与目标的偏差，优先选择缺口最大的模式
- B 模式中：第二角色固定为 Doro（粉狗）
- 本集主题天然适合 Doro 时优先选择

### 第二角色参考图规则

- 角色详细设定见「共享参数模板.md」§第二角色池
- 操作卡中需标记「⚠️ 第二角色：[角色名]参考图」
- 第二角色参考图上传统一使用「全能参考」模式
- 如该角色首次出场且无现成素材，需提前文生图生成参考图

## 二、📋 第一场景生成操作卡（强制第一项）

> ⚠️ AI 替用户做判断，直接告诉用户哪些需要准备、哪些不需要。**不要把判断甩回给用户。**

### 格式模板
```markdown
| 分类 | 检查项 | 你需要做什么 |
|------|--------|-------------|
| 🔴 必须 | 👤 角色参考图 | 上传标准角色图（固定前提） |
| 🟡 建议 | 🏙 背景参考图 | [场景名]：[判断：AI原生可画→"无需，提示词描述即可" / 场景特殊→"建议准备，理由：..."] |
| 🟡 建议 | 🖼 需参考图的道具 | [列出+理由] 或 **"全部道具AI原生可画，无需准备参考图"** |
| 🟢 无需 | ✍️ 仅需描述的道具 | [列出本场景出现的通用日用品/家具/食物等] |
| ⚠️ 提醒 | 🔄 跨场景道具 | [列出两场景都出现的道具，提醒第二场景生成时画面可能不一致] |
| ⚠️ 提醒 | 🏃 动作复杂度 | [普通日常→"无风险" / 快速奔跑/倒地/大幅度旋转→"提醒xx段动作幅度大，生成可能降质"] |
| 🔴 必须 | 🎬 开场首帧 | [首帧画面描述] + [若为道具特写切入且该道具无参考图→⚠️"此道具无参考图，AI可能画不准，建议改为全景建立再推近"] |
| 🔴 必须 | 🔚 尾帧截图 | [详细描述12s最后一帧：角色表情+道具状态+光线+情绪锚点] |
```

### 道具分级规则 ⚠️

AI 必须根据道具的"AI认知难度"替用户做判断，分为三级：

| 级别 | 判断标准 | 用户操作 | 示例 |
|------|----------|----------|------|
| 🖼 需参考图 | AI 训练数据中罕见的特定造型物品 | 需准备参考图上传至「全能参考」 | 复古绿色小电风扇、断裂的浅绿色螺旋蚊香盘、复古黄铜台灯 |
| ✍️ 仅需描述 | 通用日用品/家具/食物，外观无歧义 | 无需准备，提示词描述即可 | 沙发、茶几、地毯、纸巾、饼干、羽毛、抱枕、遥控器、杯子 |
| ⚠️ 跨场景 | 两个场景都出现 | 无需额外准备参考图，但第二场景操作卡需标记一致性风险 | 同一张茶几（第一场景完好→第二场景可能有变化） |

### 尾帧截图描述要求 ⚠️
尾帧是连接两段视频的关键桥梁，描述必须包含：
1. **角色**：此刻的表情（定格在什么情绪）、姿势、在画面中的位置
2. **道具**：关键道具的最终状态（如"断裂的蚊香盘散落在茶几上"）
3. **光线**：此帧的光线氛围（如"午后暖光从左侧窗户斜射"）
4. **情绪锚点**：一句话说明为什么要截这一帧（如"绝望心虚的表情是第二场景反转的起点"）

## 三、📋 第二场景生成操作卡（强制第二项）

> ⚠️ 生成第二段视频（12-24s）前的补充清单。AI 替用户判断。

### 格式模板
```markdown
| 分类 | 检查项 | 你需要做什么 |
|------|--------|-------------|
| 🔴 必须 | 🔗 尾帧 | 使用第一场景尾帧截图作为第二场景首帧参考 |
| 🔴 必须 | 👤 角色变体 | [无变化→"沿用第一场景角色图" / 有变化→列出具体变化+是否需要变体参考图] |
| 🟡 建议 | 🏙 背景参考图 | [同场景→"无需，尾帧已锁定场景" / 换场景→判断AI是否可画] |
| 🟡 建议 | 🖼 需参考图的新道具 | [列出第一场景未出现的新道具+理由] 或 **"无新道具需参考图"** |
| 🟢 无需 | ✍️ 仅需描述的道具 | [列出本场景新出现的通用日用品] |
| ⚠️ 提醒 | 🔄 跨场景一致性 | [引用第一场景卡跨场景道具，说明尾帧已锁定部分状态，但仍需注意xx道具可能画得不一样] |
| 🔴 必须 | 🎬 转场方式 | [直接切换/黑场x秒/特效] + [从尾帧到开场的过渡理由] |
```

### 角色变体判断规则 ⚠️
AI 必须判断角色外观是否发生变化，并替用户决定是否需要变体参考图：

| 变化类型 | 是否需要变体参考图 | 示例 |
|----------|-------------------|------|
| 轻微表情/姿势变化 | ❌ 不需要 | 从开心→惊讶 |
| 发型变化（呆毛状态） | ⚠️ 视程度而定 | 呆毛微翘→不需要；呆毛炸成蒲公英→**建议准备** |
| 服装/配饰变化 | ⚠️ 视程度而定 | 戴了个发卡→不需要；换了整套衣服→**建议准备** |
| 身体覆盖物（面粉/泡沫） | 🖼 需要 | 浑身沾满面粉→需要变体参考图 |

### 铁律（两张操作卡通用）
- **AI 替用户判断，直接给出结论**，不得写"根据实际情况""待定""用户自行判断"
- 不得把判断甩回给用户（如"每个道具是否需要参考图请用户自行决定"）
- 操作卡内容必须与内部构思的剧本一致
- 两张操作卡已覆盖全部素材需求，**不再单独输出素材需求清单**


> ⚠️ **铁律重申**：每个场景标题后紧跟角色铁律（不用 blockquote），用户可直接整段复制到即梦。

### 密度铁律 ⚠️
| 段落 | 字数要求 |
|------|----------|
| 每段 3s | **180-280 汉字** |
| 收尾段 | **200-300 汉字** |

### 每段必须包含六大要素（缺一不可）：
1. **场景氛围**：开头写环境/光线/气氛
2. **具体肢体动作**：哪个部位、怎么动、快慢、幅度
3. **面部表情变化**：眼睛/眉毛/嘴角/腮红/呆毛的具体变化
4. **拟声词**：咕嘎语（见第五节）
5. **道具/环境互动细节**
6. **光线/环境细节**

### 写作风格要求
- 像写叙事散文一样展开每个 3 秒
- 不缩写、不概括、不跳跃
- 每段以"日系萌圆暖柔handheld。"开头
- 每段以"natural motion，画面流畅不抖动，面部清晰不变形，人体结构正常，cinematic 4K quality, 电影质感，无模糊无闪烁。"结尾

## 五、咕嘎语言系统（纯拟声词，禁止人类语言）

| 拟声词 | 发声特征 | 情绪/用途 |
|--------|---------|-----------|
| 咕？ | 短轻声 | 疑惑/好奇/试探 |
| 咕咕！ | 短促二连 | 开心/兴奋/新奇 |
| 嘎！ | 短促炸裂 | 惊吓/慌张/紧张 |
| 嘎嘎嘎 | 连续发声 | 得意/大笑 |
| 咕嘎 | 混合声 | 复杂情绪 |
| 咕……嘎…… | 拖长渐弱 | 委屈/失落/无助 |
| 咕噜噜 | 含混低沉 | 满足/舒服/幸福 |
| 嘎嘎 | 急促短促 | 着急/慌乱 |

**铁律**：全程禁用人类语言台词，所有发声必须是以上拟声词变体。

## 七、角色一致性铁律

| 身体部位 | 设定 | 常见错误 |
|----------|------|----------|
| 上肢 | 鹅黄鳍状短翅膀（企鹅鳍），无手指/手掌/指关节 | ❌ 画出手指/手 | 
| 面部 | 人形日系动漫脸（翠绿大眼、鹅黄小嘴、人类五官） | ❌ 画成动物脸/鸟脸 |
| 体型 | 1:2 头身比 Q版，毛茸茸黑白连体睡衣 | ❌ 成人比例 |
| 头部 | 齐刘海碎发、黑色波波头、头顶呆毛 | ❌ 无呆毛 |
| 脸颊 | 粉色腮红（情绪变化时扩散/加深） | ❌ 无腮红 |
| 脚 | 鹅黄色蹼足 | — |
| 语言 | 仅 咕嘎 拟声词 | ❌ 人类语言台词 |

### 第二角色铁律 ⚠️

> 📎 详细设定见「共享参数模板.md」§第二角色池；此处只记录**提示词中必须锚定的无手铁律**。

| 角色 | 体型 | 手/肢 | 提示词锚点 |
|------|------|-------|-----------|
| 🤖 Doro | 四足短腿Q版粉狗，比妹妹小一圈 | **无手**，用身体蹭/头顶/翻滚表达 | "four-legged chibi, no hands, X-shaped pink facial marking" |
| ✋ 人类大手 | 温暖修长，五指分明 | **正常人类手**，这是唯一允许出现手指的角色 | "warm human hand, slender fingers, gentle gesture" |

**关键区分记忆法**：
- 妹妹 = 鳍状翅膀无手指
- 人类的手 = 正常五指（唯一例外）
- Doro = 四足粉狗无手

## 八、故事创作规则

1. **新主题**：不能与已用主题重复（参考运行时提供的已用主题列表）
2. **故事类型**：日常生活 + 笑点 + 反转
3. **结构**：内部构思 8 段故事（0-12s 第一场景 + 12-24s 第二场景），但不输出分镜描述
4. **第二场景**：秒数从 0 重新计数，用 `---` 分隔两场景
5. **模式选择**：优先补 A/B/C 缺口，第二角色优先用出场最少的（见 §一-附智能轮换规则）
6. **第二角色不出场时**：不得强行在故事中提及或暗示任何第二角色，保持故事纯粹
